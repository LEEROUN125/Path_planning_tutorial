# Path Planning Tutorial

A comprehensive path planning tutorial for mobile robots, featuring multiple algorithms with real-time visual simulation. This project demonstrates core autonomous navigation concepts, bridging the gap between geometric algorithms and kinematic robot constraints.

## ✨ Features

- **Multiple Path Planning Algorithms**
  - A* (A-Star): Grid-based optimal path search with corner-cutting prevention.
  - Dijkstra's Algorithm: Uninformed grid-based search.
  - RRT (Rapidly-exploring Random Tree): Sampling-based planning with edge-collision checking.
  - RRT* (Optimal RRT): Asymptotic optimal sampling with safe rewiring.

- **Visual Simulation**
  - Real-time robot movement and path following.
  - **Search Tree Visualization**: View the candidate branches generated by RRT/RRT*.
  - **Dynamic Grid Overlay**: Visualizes the actual A* grid resolution.
  - Path display with transparency for Ground Truth comparison.

- **Robot Model**
  - **Bicycle Kinematics**: Non-holonomic steering behavior (front-wheel steering, rear-axle pivot).
  - **Collision Awareness**: Uses a circumcircle radius (`0.65m`) to ensure the robot's corners do not scrape walls during wide turns.
  - Smooth path resampling for dense waypoint tracking.

- **Environment**
  - Random obstacle and internal wall generation.
  - **Configuration Space (C-Space)**: Obstacles are mathematically inflated by the robot's radius to ensure completely safe passageways.

## 🚀 Getting Started

### Prerequisites

- **C++20 compatible compiler** (g++ or clang++)
- **CMake 3.16+**
- **SFML 2.5+** or **SFML 3.0+**
- **Docker** (optional, for containerized development)

### Quick Start (Docker)

```bash
# 1. Allow X11 access
xhost +local:docker

# 2. Build and start container
cd ~/open_tutorial_ws/Path_planning_tutorial
docker compose -f docker/docker-compose.yml up --build -d

# 3. Access container
docker exec -it path_planning_sim bash

# 4. Run the simulation (already built in Dockerfile)
cd /project/build
./bin/path_planning
```

### Customization

1. Map configuration

```
// Change the grid resolution (higher = finer paths, but slower A*)
static constexpr int grid_cols = 100;
static constexpr int grid_rows = 50;

// Ensure min_gap is strictly larger than (Robot::radius * 2)
// so the robot can physically pass through doors!
float min_gap = 2.0F;
float safety_padding = 0.15F; // Buffer distance from walls
```

2. Robot parameters

Edit include/Robot.h

```
// Robot dimensions
static constexpr float length = 0.6F;
static constexpr float width = 0.4F;

// CRITICAL: The collision radius must enclose the entire robot
// based on its turning center (Rear axle for Bicycle model)
static constexpr float radius = 0.65F;

// Motion parameters
static constexpr float linear_velocity = 1.5F;      // m/s
static constexpr float max_angular_velocity = 2.0F; // rad/s
```

3. Algorithm Parameters

For example, RRT, modify in include/RRTPlanner.h

```
float stepSize = 0.3f;      // Maximum step size
float goalBias = 0.1f;      // Probability of sampling goal
int maxIterations = 10000;  // Maximum tree nodes
```
